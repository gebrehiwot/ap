OOP basics review

classes · objects · pillars · syntax · memory

Object-Oriented Programming organizes code around objects (data + behavior). Java is class‑based — you define blueprints, then create instances.

🧱 the four pillars

📦

Encapsulation

Bundle data (fields) and methods; hide internal state using private. Access via getters/setters.

public class Person { private String name; // hidden public String getName() { return name; } }
🎭

Abstraction

Hide complex implementation, show only essential features. Achieved via abstract classes or interfaces.

interface Vehicle { void move(); // what, not how } class Car implements Vehicle { public void move() { /* ... */ } }
🔄

Inheritance

Create new classes based on existing ones (extends). Reuse fields/methods and add new features.

class Animal { } class Dog extends Animal { }
✨

Polymorphism

“many forms” — same method name, different behavior (overloading / overriding).

void print(String s) { } void print(int i) { } // overload
🏷️ class · object

A class is a blueprint. An object is an instance built from that blueprint — lives in heap memory.

📋 Class

  • defines fields/methods
  • loaded once
  • template

🧩 Object

  • has state (values)
  • created with new
  • multiple instances
// Class definition class Student { String name; // instance var } // Objects Student s1 = new Student(); Student s2 = new Student();
building blocks

⚙️ fields · methods · constructors

📌 fields

variables inside a class — hold state. can be primitive or reference.

📌 methods

operations/behavior. can return values, accept parameters.

📌 constructors

special method called when object is instantiated (new). initialize state.

public class Book { String title; // field int pages; // constructor public Book(String title, int pages) { this.title = title; this.pages = pages; } // method public void display() { System.out.println(title + " (" + pages + ")"); } }
static context

🧠 static vs instance

static belongs to the class, not object. shared across all instances.

instance belongs to each object (different values possible).

class Counter { static int total = 0; // one copy for all int id; // each object gets own id Counter() { total++; } }
reference

🔍 this keyword

refers to the current object. used to distinguish fields from parameters, or call another constructor.

public class Rectangle { int w, h; public Rectangle(int w, int h) { this.w = w; // ← field = parameter this.h = h; } public Rectangle() { this(1,1); // calls above constructor } }

⚡ overloading · overriding

➕ overloading

same method name, different parameters (within same class). compile‑time.

void add(int a) { } void add(int a, int b) { }

🔁 overriding

subclass redefines a method from parent. runtime polymorphism.

@Override public String toString() { return "My version"; }
JVM memory

🧠 stack · heap

stack — primitives and references (local variables). method calls go here.

heap — all objects created with new live here. garbage collected.

Person p = new Person(); // p (reference) on stack → object on heap
