🧱 1. where it begins: class & object

Class = blueprint. Object = real thing built from that blueprint.

blueprint

class Car

class Car { String color; void drive() { System.out.println("vroom"); } }
instances

objects

Car c1 = new Car(); c1.color = "red"; Car c2 = new Car(); c2.color = "blue";

🔒 2. encapsulation

Hide data, expose methods. Use private so nobody messes with internal state directly.

public class BankAccount { private double balance; // hidden public void deposit(double amount) { if(amount > 0) balance += amount; } public double getBalance() { return balance; } }
✏️ draw a wall around "balance". only deposit() and getBalance() are doors.
🌿 3. inheritance

Child gets parent's fields/methods. extends keyword. reuse!

parent

class Animal { void eat() { System.out.println("eating"); } }

child

class Dog extends Animal { void bark() { } } // Dog can eat() too

➕ super() calls parent constructor.

class Vehicle { String brand; Vehicle(String b) { brand = b; } } class Car extends Vehicle { Car(String b) { super(b); } }
✏️ arrow from Dog to Animal — "is-a" relationship. Dog inherits eat().
🌀 4. polymorphism

same method, different behavior – overloading (compile time) & overriding (runtime).

⚙️ overloading (same class)

class Math { int add(int a, int b) { return a+b; } int add(int a, int b, int c) { return a+b+c; } }

✨ overriding (subclass redefines)

class Animal { void sound() { } } class Cat extends Animal { @Override void sound() { System.out.println("meow"); } }

polymorphic array:

Animal[] animals = {new Cat(), new Dog()}; for (Animal a : animals) { a.sound(); // which sound? → JVM decides at runtime }
✏️ draw animals array pointing to Cat object and Dog object. then draw arrow to sound() method versions.
📄 5. interface

Contract: what a class must do, but not how. implements.

interface Flyable { void fly(); // abstract (no body) } class Bird implements Flyable { public void fly() { System.out.println("flap flap"); } }

📌 multiple interfaces

interface Swimmer { void swim(); } interface Runner { void run(); } class Athlete implements Swimmer, Runner { public void swim() { } public void run() { } }
✏️ interface = sticky note of required methods. class signs contract.
⚡ all together now

Penguin extends Animal and implements Swimmer, Walker.

interface Swimmer { void swim(); } interface Walker { void walk(); } class Animal { String name; Animal(String name) { this.name = name; } void eat() { System.out.println(name + " eats"); } } class Penguin extends Animal implements Swimmer, Walker { Penguin(String name) { super(name); } public void swim() { System.out.println(name + " swims"); } public void walk() { System.out.println(name + " waddles"); } } // polymorphic usage: Swimmer s = new Penguin("Pingu"); s.swim(); // ✅ Walker w = new Penguin("Pingu"); w.walk(); // ✅
✏️ big drawing: Penguin gets eat() from Animal, plus swim() from Swimmer, walk() from Walker.
