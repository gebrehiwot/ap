☕ Java OOP · basics

class · object · encapsulation · interface · inheritance · polymorphism

📌 1. class

A class is a blueprint or template. It describes what data (fields) and behavior (methods) an object will have.
Think of it like a cookie cutter — defines shape, but is not a cookie itself.

// Class definition (blueprint) class Car { String color; // field (data) int speed; void accelerate() { // method (behavior) speed = speed + 10; } }

🔹 2. object

An object is an instance of a class. Created with new. It lives in memory and holds actual values.

example

🧩 object 1

Car myCar = new Car(); myCar.color = "red"; myCar.speed = 0;

🧩 object 2

Car momsCar = new Car(); momsCar.color = "blue"; momsCar.speed = 20;

myCar and momsCar are two separate objects, each with its own color and speed.

🔒 3. encapsulation

Encapsulation = hiding internal data and only allowing controlled access.
Use private fields + public getters/setters. Protects from unwanted changes.

public class BankAccount { private double balance; // hidden public double getBalance() { return balance; } public void deposit(double amount) { if(amount > 0) { balance = balance + amount; } } }

📋 4. interface

An interface is a contract: it lists methods that a class must implement.
Only method signatures (no body). Focus on what, not how.

interface Drawable { void draw(); // abstract method (no body) } class Circle implements Drawable { public void draw() { System.out.println("⚪ drawing circle"); } } class Square implements Drawable { public void draw() { System.out.println("⬛ drawing square"); } }

🌿 5. inheritance

Inheritance = one class acquires fields/methods from another (extends). Promotes reuse.
Parent class (superclass) → child class (subclass). Java supports single inheritance.

class Animal { void eat() { System.out.println("eating..."); } } class Dog extends Animal { void bark() { System.out.println("woof!"); } } // Dog inherits eat() from Animal

🌀 6. polymorphism

Polymorphism = "many forms". Same method name, different behavior.

⚡ compile‑time (overloading)

Same class, same method name, different parameters.

class MathUtils { int add(int a, int b) { return a+b; } int add(int a, int b, int c) { return a+b+c; } }

✨ runtime (overriding)

Subclass redefines a parent method.

Animal a = new Dog(); a.eat(); // Dog's version if overridden
