🔹 1. inheritance deep dive

Inheritance = class acquires fields/methods from another (parent). Java uses extends. Creates an "is‑a" relationship.

superclass / parent

Animal (base)

class Animal { String name; void eat() { System.out.println("eating..."); } }
subclass / child

Dog extends Animal

class Dog extends Animal { void bark() { System.out.println("woof!"); } }
key point

✔️ reuse: Dog automatically has name and eat() without rewriting.
✔️ can add new members (bark()) or override existing ones.
✔️ Java supports single inheritance (one direct parent).

📚 super keyword · constructor chaining

Use super() to call parent constructor (must be first statement).

class Vehicle { String brand; Vehicle(String brand) { this.brand = brand; } } class Car extends Vehicle { int doors; Car(String brand, int doors) { super(brand); // call parent constructor this.doors = doors; } }

🧬 method overriding + @Override

Child redefines a parent method. Same signature, different implementation.

parent

class Animal { void sound() { System.out.println("generic sound"); } }

child (overrides)

class Cat extends Animal { @Override void sound() { System.out.println("meow"); } }

🌀 2. polymorphism (many forms)

Polymorphism = same method name behaves differently based on object. Two types: compile‑time (overloading) & runtime (overriding).

⚙️ overloading

Same class, same method name, different parameters. Compiler decides at compile time.

class Printer { void print(int i) { System.out.println(i); } void print(String s) { System.out.println(s); } void print(int i, String s) { } }

✨ overriding (runtime)

Subclass provides specific implementation. JVM chooses method based on actual object type (dynamic dispatch).

Animal a = new Cat(); a.sound(); // "meow" (Cat's version)
🔁 polymorphic collections

Store different objects as parent type, call overridden methods — all invoke the correct subclass version.

Animal[] pets = {new Dog(), new Cat(), new Animal()}; for (Animal a : pets) { a.sound(); // Dog's bark? Cat's meow? Animal's generic? }

📐 3. interfaces · contract · design

An interface defines a contract: what methods a class must implement. No method bodies (before Java 8 default methods). Supports multiple inheritance of type.

interface definition
interface Flyable { void fly(); // abstract }
implementation
class Bird implements Flyable { public void fly() { System.out.println("bird flying"); } }

🧩 multiple interfaces

Java class can implement multiple interfaces (unlike inheritance).

interface Swimmer { void swim(); } interface Runner { void run(); } class Athlete implements Swimmer, Runner { public void swim() { } public void run() { } }
🔀 interface vs abstract class

Interface = pure contract (what). Abstract class = partial implementation + state. Use interface for capability/mixin.

🌐 default methods (Java 8+)

Interfaces can have default methods with body — backward compatibility.

interface Flyable { default void land() { System.out.println("landing..."); } }

🧪 all together: inheritance + interface + polymorphism

Class Penguin extends Animal and implements Swimmer + Walker.

interface Swimmer { void swim(); } interface Walker { void walk(); } class Animal { String name; Animal(String name) { this.name = name; } void eat() { System.out.println(name + " eats"); } } class Penguin extends Animal implements Swimmer, Walker { Penguin(String name) { super(name); } @Override public void swim() { System.out.println(name + " glides through water"); } @Override public void walk() { System.out.println(name + " waddles"); } } // polymorphic usage Swimmer s = new Penguin("Pingu"); s.swim(); // ✓ Walker w = new Penguin("Pingu"); w.walk(); // ✓
