1. class & object

start here Class = blueprint · Object = instance (the real thing).

📐 class

class Student { String name; int age; void study() { System.out.println(name + " is studying"); } }

🧩 objects

Student s1 = new Student(); s1.name = "Alice"; Student s2 = new Student(); s2.name = "Bob";
✏️ draw: class as a rubber stamp → stamped out objects (s1, s2). each has own name.
2. encapsulation

Hide data, control access — use private + getters/setters.

public class BankAccount { private double balance; // 🔒 hidden public void deposit(double amt) { if(amt > 0) balance += amt; } public double getBalance() { return balance; } }
✏️ draw a protective wall around 'balance'. only deposit() and getBalance() are gates.
3. inheritance

extends — child gets parent's fields/methods. "is‑a" relationship.

parent

class Animal { void eat() { System.out.println("eating..."); } }

child

class Dog extends Animal { void bark() { } } // Dog can eat() too

super() calls parent constructor.

class Vehicle { String brand; Vehicle(String b) { brand = b; } } class Car extends Vehicle { Car(String b) { super(b); } }
✏️ arrow from Dog to Animal — inherits eat(). add super() link.
4. polymorphism

many forms — same method name, different behavior.

⚙️ overloading (compile‑time)

class Printer { void print(int i) { } void print(String s) { } void print(int i, String s) { } }

✨ overriding (runtime)

class Cat extends Animal { @Override void eat() { System.out.println("cat eats"); } }

polymorphic array example:

Animal[] animals = {new Dog(), new Cat()}; for (Animal a : animals) { a.eat(); // JVM picks right version }
✏️ draw array pointing to Dog and Cat objects, then arrows to overridden eat() methods.
5. interface

contract — what a class must do, but not how. implements

interface Flyable { void fly(); // abstract (no body) } class Bird implements Flyable { public void fly() { System.out.println("bird flies"); } }

multiple interfaces

interface Swimmer { void swim(); } interface Runner { void run(); } class Triathlete implements Swimmer, Runner { public void swim() { } public void run() { } }
✏️ interface = sticky note of required methods. class signs contract.
6. abstract class

partial implementation — can have both abstract & concrete methods. cannot instantiate.

abstract class Shape { abstract double area(); // must be implemented by subclass void display() { System.out.println("display shape"); } } class Circle extends Shape { double area() { return 3.14 * r * r; } }
✏️ abstract = ghost class (can't new). concrete subclass fills in area().
7. exceptions

handle runtime errors — try, catch, finally, throws

checked exception

// must handle or declare try { Thread.sleep(1000); } catch (InterruptedException e) { e.printStackTrace(); }

unchecked (RuntimeException)

int x = 10 / 0; // ArithmeticException

try-catch-finally

try { int[] arr = new int[2]; System.out.println(arr[5]); } catch (ArrayIndexOutOfBoundsException e) { System.out.println("index out of bound!"); } finally { System.out.println("always runs"); }

throws / throw

void readFile() throws IOException { if (someCondition) { throw new IOException("file missing"); } }
✏️ draw try block as protective bubble. if exception → catch net. finally runs regardless.
